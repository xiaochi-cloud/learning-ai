# 提示词的 4 个核心要素

> 🎯 一句话核心：**好提示词 = 清晰任务 + 充分背景 + 具体格式 + 明确约束**

---

## 📋 3 个关键要点

1. **任务清晰** — AI 知道要做什么
2. **背景充分** — AI 理解为什么做
3. **格式具体** — AI 知道做成什么样

---

## 💡 详细说明

### 要素 1：任务清晰（做什么）

告诉 AI 具体要完成什么任务，越具体越好。

**❌ 模糊：**
```
写个代码
```

**✅ 清晰：**
```
用 Python 写一个函数，输入文件路径，返回文件行数
```

**对比分析：**

| 维度 | 模糊 | 清晰 |
|------|------|------|
| 语言 | 未指定 | Python |
| 功能 | 不明确 | 统计行数 |
| 输入 | 未知 | 文件路径 |
| 输出 | 未知 | 行数（整数） |

---

### 要素 2：背景充分（为什么）

给 AI 必要的上下文，帮助它理解任务场景。

**❌ 无背景：**
```
分析这个数据
```

**✅ 有背景：**
```
我有一份销售数据，包含日期、产品、销售额三列。
我是市场分析师，需要找出销售额最高的产品和时间段。
请帮我分析数据趋势，给出 3 个关键发现。
```

**背景包含：**
- 数据来源和结构
- 你的角色和目标
- 期望的分析角度

---

### 要素 3：格式具体（什么样）

明确告诉 AI 你期望的输出格式。

**❌ 无格式：**
```
介绍一下 Python
```

**✅ 有格式：**
```
请用以下格式介绍 Python：

【一句话定义】不超过 30 字
【核心特点】列出 5 个 bullet points
【适用场景】列出 3 个主要用途
【学习建议】给初学者的 1 条建议
```

**输出示例：**

```
【一句话定义】
Python 是简洁易读的高级编程语言，适合多种应用场景。

【核心特点】
• 语法简洁，接近自然语言
• 丰富的标准库和第三方库
• 跨平台，支持多种操作系统
• 支持多种编程范式
• 活跃的开发社区

【适用场景】
• Web 开发（Django, Flask）
• 数据分析和机器学习
• 自动化脚本

【学习建议】
从基础语法开始，边学边做小项目，不要只看不练。
```

---

### 要素 4：约束明确（不要什么）

说明限制条件和禁忌，避免 AI 跑偏。

**约束类型：**

| 类型 | 示例 |
|------|------|
| 长度约束 | "不超过 200 字" |
| 风格约束 | "用正式语气" |
| 内容约束 | "不要使用专业术语" |
| 格式约束 | "用表格展示" |
| 时间约束 | "基于 2024 年之前的知识" |

## 📝 完整示例

### 示例：写一个邮件回复助手

**❌ 差提示词：**
```
帮我写邮件回复
```

**✅ 好提示词（4 要素齐全）：**

```
【任务】
帮我写一封邮件回复，回复客户的询价邮件。

【背景】
- 我是销售经理，对方是潜在客户
- 他们询问了我们产品的价格和交货期
- 我们产品价格是 10000 元，交货期 2 周
- 希望语气专业但友好，促进合作

【格式】
请按以下结构：
1. 称呼
2. 感谢询价
3. 产品价格
4. 交货期
5. 下一步建议
6. 结束语

【约束】
- 字数 200-300 字
- 不要使用过于销售化的语言
- 包含我的联系方式（电话：xxx，邮箱：xxx）
- 用中文
```

---

## ⚠️ 常见错误

| 错误 | 表现 | 正确做法 |
|------|------|----------|
| 任务模糊 | "处理一下" | 明确具体任务 |
| 背景缺失 | 直接问问题 | 先说明场景 |
| 格式随意 | "随便写" | 指定输出结构 |
| 约束遗漏 | 没有长度限制 | 说明要求 |

## 🎯 小练习

优化以下提示词（应用 4 要素）：

```
分析一下这篇文章
```

**参考答案：**

```
【任务】
分析以下文章的核心观点和论证结构。

【背景】
我是大学生，需要写读后感。文章是关于 AI 对就业影响的评论。

【格式】
请按以下结构输出：
1. 核心观点（1 句话）
2. 主要论据（3 个 bullet points）
3. 论证方法（举例/对比/数据等）
4. 我的评价（100 字以内）

【约束】
- 总字数 300-400 字
- 用通俗易懂的语言
- 不要引用原文
```
