# 输出格式控制的 5 种方法

> 🎯 一句话核心：**格式越具体，输出越可控**

---

## 📋 5 种方法概览

| 方法 | 适用场景 | 控制力 |
|------|----------|--------|
| 1. 明确描述 | 简单格式 | ⭐⭐⭐ |
| 2. 提供示例 | 复杂格式 | ⭐⭐⭐⭐ |
| 3. 模板填充 | 固定格式 | ⭐⭐⭐⭐⭐ |
| 4. 结构化指令 | 数据格式 | ⭐⭐⭐⭐⭐ |
| 5. 约束强化 | 严格格式 | ⭐⭐⭐⭐⭐ |

---

## 方法 1：明确描述

### 适用场景
简单格式要求，如表格、列表、段落等。

### 示例

**表格：**
```
请用表格形式展示，包含以下列：
- 产品名称
- 价格
- 销量
- 增长率

按销量降序排列。
```

**列表：**
```
请用编号列表列出 5 个要点，
每个要点包含：
- 标题（5 字以内）
- 说明（1-2 句）
```

**段落：**
```
请分 3 段回答：
第 1 段：问题定义
第 2 段：分析方法
第 3 段：结论建议

每段 100-150 字。
```

---

## 方法 2：提供示例

### 适用场景
复杂格式或自定义格式。

### 示例

**JSON 格式：**
```
请输出 JSON 格式，结构如下：

```json
{
  "summary": "一句话总结",
  "analysis": {
    "strengths": ["优势 1", "优势 2"],
    "weaknesses": ["劣势 1", "劣势 2"]
  },
  "recommendations": [
    {"priority": "高", "action": "具体行动"},
    {"priority": "中", "action": "具体行动"}
  ]
}
```

严格遵循以上结构，不要添加额外字段。
```

**报告格式：**
```
请按以下格式输出报告：

================================
           分析报告
================================

【基本信息】
- 分析对象：
- 分析日期：
- 分析师：AI 助手

【核心结论】
[1 句话总结]

【详细分析】
1. 方面 1
   - 现状：
   - 问题：
   - 建议：

2. 方面 2
   - 现状：
   - 问题：
   - 建议：

【下一步行动】
- [ ] 行动 1
- [ ] 行动 2
- [ ] 行动 3

================================
```

## 方法 3：模板填充

### 适用场景
固定格式，批量生成。

### 示例

**邮件模板：**
```
请填充以下邮件模板：

---
主题：[根据内容填写]

尊敬的 [客户姓名]：

感谢您 [具体事项]。

关于您询问的 [问题]，我们的回复如下：

[回复内容，3-5 句]

如果您有任何其他问题，请随时联系我们。

此致
敬礼

[我的姓名]
[职位]
[联系方式]
---

背景信息：
- 客户姓名：张三
- 询问事项：产品价格和交货期
- 价格：10000 元
- 交货期：2 周
```

**代码注释模板：**
```
请为以下函数添加注释，使用此模板：

```python
def function_name(params):
    """
    【功能】[一句话描述]
    
    【参数】
    - param1: [类型] [说明]
    - param2: [类型] [说明]
    
    【返回】
    - [类型] [说明]
    
    【示例】
    >>> function_name(example_params)
    >>> example_output
    
    【注意】
    - [注意事项 1]
    - [注意事项 2]
    """
    # 函数实现
```
```

## 方法 4：结构化指令

### 适用场景
数据格式、编程输出。

### 示例

**CSV 格式：**
```
请输出 CSV 格式数据：

列定义：
1. id (整数，自增)
2. name (字符串，产品名)
3. price (浮点数，2 位小数)
4. stock (整数，库存量)

要求：
- 第一行是表头
- 用逗号分隔
- 字符串用双引号包裹
- 生成 10 条示例数据
```

**Markdown 格式：**
```
请用 Markdown 格式输出，结构如下：

# [标题]

> 摘要：[1-2 句核心内容]

## 1. 第一部分
[内容]

### 1.1 子标题
[内容]

## 2. 第二部分
[内容]

## 总结
[总结内容]

---
*最后更新：[日期]*
```

## 方法 5：约束强化

### 适用场景
严格要求格式，不允许偏差。

### 示例

**开头 + 结尾双重约束：**
```
【重要】请严格按照以下格式输出，不要添加任何额外内容：

[格式要求]
- 只输出 JSON
- 不要 Markdown 标记
- 不要解释说明

[任务内容]
分析以下数据...

【再次强调】
只输出纯 JSON，不要任何其他内容。
```

**负面约束：**
```
请生成产品描述，要求：

✅ 必须：
- 标题 10-15 字
- 正文 3 段，每段 50-80 字
- 包含 3 个核心卖点

❌ 禁止：
- 不要使用感叹号
- 不要夸张用语（如"最好"、"第一"）
- 不要添加价格信息
- 不要输出任何解释
```

## ⚠️ 常见错误

| 错误 | 表现 | 改进 |
|------|------|------|
| 格式模糊 | "整理一下" | 具体格式描述 |
| 示例缺失 | "用 JSON"但无结构 | 提供完整示例 |
| 约束不足 | 没有负面约束 | 添加"禁止"项 |
| 位置不当 | 格式要求在中间 | 放在开头 + 结尾 |

## 🎯 小练习

为以下任务选择合适的格式控制方法：

**任务 1：** 生成产品数据（100 条）

**答案：** 方法 4（结构化指令）+ 方法 3（模板填充）
```
请生成 100 条产品数据，CSV 格式：
列：id,name,price,stock,category
[id 自增，name 合理，price 2 位小数，stock 整数，category 从 [电子/服装/食品] 选]
```

**任务 2：** 写一封标准商务邮件

**答案：** 方法 3（模板填充）
```
请使用标准商务邮件模板，填充以下内容...
```

**任务 3：** 输出分析结论（要求严格）

**答案：** 方法 5（约束强化）
```
【重要】只输出以下 3 部分，不要任何其他内容：
1. 核心结论（1 句）
2. 关键发现（3 个 bullet points）
3. 建议（2 条）
```
