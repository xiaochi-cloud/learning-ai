# Claude 3 系列模型对比

> 🎯 一句话核心：**Haiku 快、Sonnet 平衡、Opus 强，按场景选择**

---

## 📋 3 个关键要点

1. **Haiku** — 速度快、成本低，适合简单任务
2. **Sonnet** — 性能平衡，适合大多数场景
3. **Opus** — 能力最强，适合复杂任务

## 🎯 选型建议

### 选择 Haiku 的场景

✅ **适合：**
- 简单问答（事实查询）
- 内容分类/标签
- 基础翻译
- 快速原型
- 高频调用（成本敏感）

**示例：**
```
"今天上海天气怎么样？"
"把这段文字分类为正面/负面"
"翻译这句话为英文"
```

❌ **不适合：**
- 复杂推理
- 专业写作
- 深度代码生成

---

### 选择 Sonnet 的场景

✅ **适合：**
- 日常对话助手
- 邮件/文章写作
- 代码生成和审查
- 数据分析
- 大多数应用场景 ⭐

**示例：**
```
"帮我写一封商务邮件"
"分析这份销售数据"
"写一个 Python 脚本处理 Excel"
"解释这个代码片段"
```

❌ **不适合：**
- 极高难度推理
- 专业级创作（需要 Opus）

---

### 选择 Opus 的场景

✅ **适合：**
- 复杂逻辑推理
- 专业级写作（论文、报告）
- 高级代码架构设计
- 深度研究分析
- 多步骤复杂任务

**示例：**
```
"分析这篇论文的创新点和局限性"
"设计一个微服务架构方案"
"解决这个复杂的数学证明"
"撰写行业研究报告"
```

❌ **不适合：**
- 简单任务（浪费钱）
- 高频调用（成本太高）

## 💰 成本对比示例

### 场景：每日处理 100 个请求

| 模型 | 单次成本 | 每日成本 | 每月成本 |
|------|----------|----------|----------|
| Haiku | $0.001 | $0.1 | $3 |
| Sonnet | $0.01 | $1 | $30 |
| Opus | $0.05 | $5 | $150 |

**建议：** 80% 用 Haiku + 15% 用 Sonnet + 5% 用 Opus = 最优性价比

## ⚠️ 常见误区

| 误区 | 事实 |
|------|------|
| Opus 一定最好 | 简单任务用 Haiku 更快更省 |
| Haiku 太笨 | 简单任务表现很好 |
| 只用一个模型 | 混合使用性价比最高 |
| 越贵越好 | 合适最重要 |

## 🎯 小练习

为以下场景选择合适模型：

**场景 1：** 客服机器人，每日 1000+ 对话

**答案：** Haiku（高频、简单对话、成本敏感）

**场景 2：** 技术博客写作，每周 2-3 篇

**答案：** Sonnet（写作质量要求高，但频率不高）

**场景 3：** 投资研究报告，每月 1 份

**答案：** Opus（高质量要求，低频率，值得用最强）
