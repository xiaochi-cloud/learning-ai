## 🎯 练习目标

通过实际对比，理解好 prompt 和差 prompt 的区别。

---

## 📝 练习 1：简单任务对比

### 任务：让 AI 介绍自己

**❌ 差 prompt：**
```
介绍一下
```

**✅ 好 prompt：**
```
请用 3 句话介绍你自己，要求：
1. 第一句说明你是谁
2. 第二句说明你能做什么
3. 第三句给用户一个使用建议
语言简洁，每句不超过 30 字。
```

**对比分析：**
- 差 prompt 太模糊，AI 不知道你想要什么
- 好 prompt 有明确结构、长度、内容要求

---

## 📝 练习 2：代码任务对比

### 任务：让 AI 写一个排序函数

**❌ 差 prompt：**
```
写个排序代码
```

**✅ 好 prompt：**
```
用 Python 写一个快速排序函数，要求：
- 函数名：quick_sort
- 输入：一个整数列表
- 输出：排序后的新列表（不修改原列表）
- 添加类型注解
- 包含 2 个测试用例
- 添加简要注释
```

**对比分析：**
- 差 prompt 没有语言、功能、质量要求
- 好 prompt 明确了所有关键信息

## 📝 练习 3：分析任务对比

### 任务：让 AI 分析一篇文章

**❌ 差 prompt：**
```
分析一下这篇文章
```

**✅ 好 prompt：**
```
请分析以下文章，输出包含：

【核心观点】用 1 句话总结
【论据要点】列出 3 个主要论据
【我的评价】从可信度、逻辑性、实用性三个维度评价（各 1 句话）

文章：
[粘贴文章内容]
```

**对比分析：**
- 差 prompt 没有分析框架
- 好 prompt 给出了清晰的输出结构

## 💡 关键收获

### 好 Prompt 的 4 个要素

```
1. 任务清晰 — 明确告诉 AI 要做什么
2. 背景充分 — 提供必要的上下文
3. 格式具体 — 说明期望的输出格式
4. 约束明确 — 长度、风格、禁忌等
```

### Prompt 迭代思维

第一次不完美很正常：

```
第一轮：写个 Python 脚本
第二轮：用 Python 写一个处理 CSV 的脚本
第三轮：用 Python 写一个处理 CSV 的脚本，
       功能是统计每行的列数，输出为 JSON
第四轮：（添加错误处理、性能要求等）
```

## 🛠️ 我的 Prompt 模板

```
# 角色设定（可选）
你是一位 [角色]，擅长 [技能]

# 任务描述
请帮我 [具体任务]

# 背景信息
[必要的上下文]

# 输出要求
- 格式：[bullet points/表格/代码等]
- 长度：[字数限制]
- 风格：[正式/ casual/技术性等]

# 约束条件
- 不要：[禁忌]
- 必须：[必须包含的内容]
```
