# [Prompt] Prompt 太长被截断怎么办

> 🎯 一句话解决：**分块发送 + 上下文引用，突破长度限制**

---

## 🔴 错误信息

```
Error: Request too large. Maximum context length is 100K tokens.
Your request: 120K tokens
```

或者（内容被截断）：

```

```

## ✅ 解决方案

### 方案 1：精简压缩（适合超出不多）

**问题：** Prompt 110K，限制 100K

**解决：**

```
原 Prompt 问题：
- 重复说明多次
- 包含无关细节
- 格式冗余

优化后：
- 删除重复内容
- 只保留关键信息
- 用简洁格式
```

**对比：**

❌ 冗余写法：
```
我希望你能够帮我分析一下这份数据，这份数据是我们公司
2024 年全年的销售数据，包含了各个区域、各个产品线、
各个月份的销售额、销售量、利润等等信息...（500 字）
```

✅ 简洁写法：
```
分析 2024 年销售数据，包含：
- 维度：区域、产品线、月份
- 指标：销售额、销售量、利润
- 目标：找出趋势和问题
```

**节省：** 80%+

### 方案 2：分块发送（适合长文档）

**场景：** 需要分析 10 万字的文档

**方法：**

```
第 1 步：发送文档概要
"我将分块发送一份文档给你，这是第 1/5 部分，
 请先不要分析，收到后回复'收到'即可。
 
 第 1 部分：[内容]"

第 2-5 步：继续发送
"这是第 2/5 部分... [内容]"

最后一步：发送分析指令
"文档已全部发送。现在请：
1. 总结核心观点
2. 分析论证结构
3. 评价优缺点"
```

**代码示例：**

```python
def send_in_chunks(document: str, chunk_size: int = 20000):
    """分块发送长文档"""
    chunks = [document[i:i+chunk_size] for i in range(0, len(document), chunk_size)]
    total = len(chunks)
    
    for idx, chunk in enumerate(chunks, 1):
        if idx < total:
            # 中间块
            prompt = f"这是第{idx}/{total}部分，请继续接收：\n\n{chunk}"
        else:
            # 最后一块 + 指令
            prompt = f"这是最后部分（{idx}/{total}）。现在请分析：\n\n{chunk}\n\n请总结核心观点并分析论证结构。"
        
        response = claude_client.messages.create(
            model="claude-sonnet-4-20250514",
            max_tokens=2048,
            messages=[{"role": "user", "content": prompt}]
        )
        
        print(f"第{idx}块发送完成")
    
    return response
```

### 方案 3：外部引用（适合超大内容）

**场景：** 需要处理的内容远超限制

**方法：**

```
1. 将内容放到外部（文件、链接）
2. Prompt 中引用关键部分
3. 让 AI 基于摘要/关键段落分析
```

**示例：**

❌ 直接发送全部：
```
[粘贴 10 万字文档] 请分析
```

✅ 引用关键部分：
```
我有一份 10 万字的产品文档（已上传到：链接/文件）

核心内容摘要：
- 产品定位：XX
- 目标用户：XX
- 核心功能：XX
- 竞争优势：XX

请基于以上摘要，帮我：
1. 评估产品定位是否清晰
2. 分析目标用户是否准确
3. 给出 3 条改进建议

如需查看完整文档细节，请告诉我具体章节，我可以单独发送。
```

## 📌 根本原因

### 为什么有长度限制？

1. **计算资源** — 处理长文本需要更多计算
2. **响应时间** — 太长会影响响应速度
3. **注意力机制** — 模型对超长文本的理解会下降

### 各模型限制

| 模型 | 上下文限制 | 约等于 |
|------|------------|--------|
| Claude 3 Sonnet | 200K | 15 万汉字 |
| Claude 3 Haiku | 200K | 15 万汉字 |
| GPT-4 Turbo | 128K | 10 万汉字 |
| GPT-4 | 8K | 6 千汉字 |

---

## 🛡️ 如何避免

### 最佳实践

1. **写之前先精简**
   - 只包含必要信息
   - 删除重复内容
   - 用简洁表达

2. **长内容分块**
   - 超过 50K 就考虑分块
   - 提前规划块结构
   - 保持块之间连贯

3. **善用摘要**
   - 先给摘要再给细节
   - 让 AI 决定是否需要细节
   - 按需发送

### Prompt 精简检查清单

- [ ] 删除了重复说明？
- [ ] 只保留关键信息？
- [ ] 用 bullet points 代替长段落？
- [ ] 删除了无关背景？
- [ ] 格式简洁？

|------|----------|------|------|
